> **Status:** Project-defining imagery and cover checkpoint  
> **Purpose:** Select the cover family, image-frame treatment, caption/source treatment, historical-photo treatment, and game-asset reuse policy.

## Lead-Design Recommendation

### Cover Structure

**Mission-title cover**

Use the Mission Title Block logic with:

- a restrained hero visual;
- supporting role metadata;
- one institutional insignia;
- clear title hierarchy.

This is the strongest ordinary default because it stays consistent with the rest of the document system while still allowing more visual presence than interior pages.

Use the hero-image cover only for:

- packet openers;
- campaign dividers;
- showcase or portfolio documents.

### Image Frame Treatment

**Thin technical frame**

Carry forward the v0.6 figure family.

Use the same restrained frame for:

- photographs;
- screenshots;
- illustrations;
- scans;
- maps.

Open images should be exceptions.

### Caption / Source Treatment

**Caption metadata band**

This should remain universal unless a special case requires otherwise.

It keeps photos, game assets, and diagrams in the same documentary system.

### Historical-Photo Treatment

**Retain authentic tonal character**

Do not flatten every archival photo into an artificially uniform grayscale.

Permitted edits:

- crop;
- rotate if necessary;
- clean borders;
- normalize legibility;
- add frame and caption.

Preserve the source's historical character where possible.

### Game-Asset Reuse Policy

**Selective framed reuse**

Game assets may appear when they directly support:

- comparison;
- explanation;
- orientation;
- evidence discussion;
- crosswalk between gameplay and lesson content.

They must be:

- clearly bounded;
- captioned;
- sourced as game assets;
- used sparingly.

## Key Imagery Principles

- An image must earn its place.
- Images do not serve as generic decoration.
- The document remains curricular first and atmospheric second.
- Real historical or scientific sources must remain visually distinct from fictional or game-derived materials.
- All images require attribution or source status.
- Screenshots and sprites never replace a clearer curriculum-original figure when one would teach better.
