# VISUAL_STYLE_GUIDE_v1.0 — Compliance Checklist

> Complete this checklist before publishing a curriculum document or packet.

## Document Identity

- [ ] Correct institution is shown.
- [ ] Correct repository insignia is used.
- [ ] SSS uses an operational mission / systems evidence archetype where appropriate.
- [ ] HHH uses an archival source / chronology / provenance archetype where appropriate.
- [ ] Shared geometry is not mistaken for insufficient institutional differentiation.

- [ ] Document role is visible.
- [ ] Campaign and case identifiers are correct.
- [ ] First page uses the Mission Title Block.
- [ ] Continuation pages use the compact header.
- [ ] Production metadata remains in the footer or cover metadata.
- [ ] Page number and total are correct.

## Page Geometry

- [ ] Page is US Letter portrait unless an exception is documented.
- [ ] Margins are 0.50 inches unless a page-specific exception is documented.
- [ ] Content aligns to the 12-column grid.
- [ ] Gutters and left edges are consistent.
- [ ] Required content remains inside printable safety.
- [ ] No overflow warning remains.
- [ ] Flexible student work areas expand into avoidable surplus space.
- [ ] No large unused region remains below a response area that could productively expand.
- [ ] A short `Page 1 of 2` has been reflowed when Page 2 content could fit.
- [ ] Short teacher/reference pages are not padded with oversized type or decoration.
- [ ] Teacher notes areas use remaining page space when they are the page's flexible final component.
- [ ] Accessible response areas do not leave avoidable blank space beneath them.

- [ ] Screen-only controls do not print.

## Typography and Color

- [ ] Inter is used for body and headings.
- [ ] JetBrains Mono is used for technical text.
- [ ] Approved size and weight ranges are followed.
- [ ] Browser reference templates use true point-equivalent CSS sizes.

- [ ] Institutional colors match v1.0.
- [ ] Grayscale hierarchy remains clear.
- [ ] Color is not the only carrier of meaning.
- [ ] Contrast is sufficient.

## Section Structure

- [ ] Recurring standardized sections use their approved Phosphor icons.
- [ ] Icons retain visible labels.
- [ ] One-off subsections do not invent arbitrary icons.
- [ ] Technical labels remain short and functional.
- [ ] The page does not box every section.

## Callouts and Status

- [ ] Priority information uses the approved left-rule callout.
- [ ] Semantic callout has icon + technical label.
- [ ] Caution, documented, inferred, fictional, and optional states are explicit.
- [ ] One callout has one primary semantic marker.
- [ ] Publication status uses the approved filled mark.
- [ ] Role label is not confused with publication status.

## Student Responses

- [ ] Ordinary response areas are blank and unruled.
- [ ] Prompt is above the response box.
- [ ] Guidance is concise.
- [ ] Response area matches expected length.
- [ ] CER uses Claim, Evidence, and Reasoning stacked boxes.
- [ ] Evidence receives enough space.
- [ ] Reasoning asks for mechanism or connection.

## Tables, Forms, and Rubrics

- [ ] Student tables use the ruled technical treatment.
- [ ] Handwriting cells are large enough.
- [ ] Checkboxes mean select-all or completion.
- [ ] Circular controls mean choose-one.
- [ ] Every control has a visible label.
- [ ] Scale endpoints are labeled.
- [ ] Analytic rubric descriptors are observable and distinct.
- [ ] Student and teacher rubric wording match.

## Teacher Output

- [ ] Teacher notes use the neutral side stripe.
- [ ] Dedicated answer-key blocks are used.
- [ ] Answer-key order mirrors student order.
- [ ] Model response allows valid alternatives.
- [ ] Common incomplete responses are identified where useful.
- [ ] Teacher content is absent from student output.
- [ ] Hidden teacher content leaves no unexplained gaps.

## Figures and Data

- [ ] Figure has a thin technical frame.
- [ ] Major elements are directly labeled.
- [ ] Orthogonal connectors are used where appropriate.
- [ ] Relationship lines are labeled when meaning matters.
- [ ] Observed, inferred, reconstructed, and fictional states are explicit.
- [ ] Caption metadata band is present.
- [ ] Source status is accurate.
- [ ] Graph axes and units are labeled.
- [ ] Graph series differ without color.
- [ ] Map geometry is sourced.
- [ ] Historical period and uncertainty are stated.

## Images and Game Assets

- [ ] Image has an instructional or documentary purpose.
- [ ] Historical photograph retains authentic tonal character.
- [ ] Crop does not alter meaning.
- [ ] Scan or excerpt is labeled.
- [ ] Adaptation or restoration is disclosed.
- [ ] Game-asset reuse is selective.
- [ ] Game source is identified.
- [ ] Game art is not presented as real evidence.
- [ ] Pixel art uses nearest-neighbor scaling.
- [ ] Image is not distorted.

## Attribution and Rights

- [ ] Creator or organization is recorded.
- [ ] Source and date are recorded where relevant.
- [ ] License or rights status is recorded.
- [ ] Adaptation is identified.
- [ ] Public-domain status has been verified.
- [ ] Generated imagery is not misrepresented.
- [ ] Full attribution is available in teacher or source documentation.

## Accessibility

- [ ] Required parallel accessible edition exists.
- [ ] Content order and identifiers match the standard edition.
- [ ] Digital reading order is logical.
- [ ] Controls are keyboard accessible.
- [ ] Focus indication is visible.
- [ ] Images have appropriate alt text.
- [ ] Complex figures have extended support.
- [ ] Transcript or translation is provided where needed.
- [ ] Alternate response modes are documented.
- [ ] Large-print output uses appropriate type and spacing.
- [ ] Enlarged accessible response areas remain clearly associated with their prompts.
- [ ] No arbitrary maximum response height prevents an accessible box from using available page area.
- [ ] Accessible handwriting, drawing, and annotation areas expand into usable surplus space.

## Print Production

- [ ] Color PDF reviewed.
- [ ] Grayscale PDF reviewed.
- [ ] Laser print tested.
- [ ] Inkjet sample tested where required.
- [ ] Low-ink output tested.
- [ ] First-generation photocopy tested.
- [ ] Second-generation photocopy tested where required.
- [ ] Duplex order and orientation tested.
- [ ] Binding and three-hole-punch safety tested.
- [ ] Pale fills survive or have structural fallback.
- [ ] Minimum rules remain visible.
- [ ] QR codes physically scan where used.
- [ ] Document does not depend on borderless printing.

## Final Release Gate

- [ ] No placeholder text or artwork remains.
- [ ] No unresolved accessibility blocker remains.
- [ ] No unresolved attribution blocker remains.
- [ ] No overflow remains.
- [ ] Compatibility metadata is correct.
- [ ] File name follows publishing convention.
- [ ] Source master is retained.
- [ ] Exported PDF matches the source master.
- [ ] Document owner approved release.
- [ ] Deviations from v1.0 are documented.
